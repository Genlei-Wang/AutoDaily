# AutoDaily 快速开始指南

## 🚀 三种使用方式

### 方式一：GitHub Actions 云端编译（最简单，推荐）

**适合：** 在 Mac 上开发，需要 Windows 编译结果

1. **上传代码到 GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git
   git push -u origin main
   ```

2. **自动编译**
   - 代码推送后，GitHub Actions 自动编译
   - 进入仓库的 `Actions` 页面
   - 等待编译完成（约 2-3 分钟）
   - 下载 `AutoDaily-Release.zip`

3. **使用**
   - 解压 ZIP 文件
   - 双击 `AutoDaily.exe` 运行

**优点：** 无需 Windows 环境，完全自动化

---

### 方式二：Windows 本地编译 + 打包脚本

**适合：** 有 Windows 电脑，需要自定义打包

1. **克隆代码到 Windows**
   ```powershell
   git clone https://github.com/YOUR_USERNAME/REPO_NAME.git
   cd REPO_NAME
   ```

2. **运行打包脚本**
   ```powershell
   cd scripts
   .\create-installer.ps1
   ```

3. **获取结果**
   - 编译好的文件在 `dist\AutoDaily-v1.0.0.zip`
   - 解压即可使用

**优点：** 可以自定义版本号和打包内容

---

### 方式三：Visual Studio 手动编译

**适合：** 需要调试和开发

1. **打开项目**
   - 双击 `AutoDaily.sln`
   - Visual Studio 自动打开

2. **编译**
   - 按 `Ctrl+Shift+B` 或
   - `生成` → `生成解决方案`

3. **运行**
   - 按 `F5` 运行调试
   - 或运行 `bin\Release\AutoDaily.exe`

**优点：** 适合开发和调试


## ❓ 常见问题

### Q: GitHub Actions 编译失败？

**A:** 
1. 检查 `.github/workflows/build.yml` 是否存在
2. 查看 Actions 页面的错误信息
3. 确保代码没有语法错误

### Q: 如何修改版本号？

**A:**
- GitHub Actions：自动使用时间戳
- 本地脚本：修改 `scripts/create-installer.ps1` 中的 `$Version` 参数

### Q: 编译产物在哪里？

**A:**
- GitHub Actions：`Actions` → 选择运行 → `Artifacts` → 下载
- 本地脚本：`dist\AutoDaily-v1.0.0.zip`
- Visual Studio：`AutoDaily\bin\Release\AutoDaily.exe`

### Q: 可以创建安装程序（.msi）吗？

**A:** 可以，但需要额外工具：
- WiX Toolset（免费）
- Inno Setup（免费）
- Advanced Installer（商业）

当前 ZIP 包已经足够使用，解压即用。
