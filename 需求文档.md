
# 背景
- 需求场景：银行、政府等单位，各个部门的职员，每天早上都要做日报，访问公司的系统（不是浏览器；电脑无法连互联网），登录，点进特定栏目，点点点，下载一个报表excel；然后打开特定的excel文件，用新数据（全选，复制），覆盖（粘贴）这部分内容，其他sheet会自动算出一些数据，这些数据要发邮件，打开邮件，点击发送。
- 目前我用几个软件组合，实现了上述场景（不一定体验好）：用Ttinytask录制用户屏幕操作，保存为exe；用system Scheduler设定时任务，每天早上自动跑。俩软件安装、使用比较复杂
- 现在我想更好的解决上述场景，计划开发一个非常轻量、易用、用户体验好的桌面软件。

# AutoDaily 产品界面设计规范 (UI/UX Specification)

## 1. 设计理念
*   **隐喻 (Metaphor)**：参考“录音笔”或“洗衣机”的控制面板。
*   **分区 (Zoning)**：将“操作区”与“设置区”视觉分离，避免误操作。
*   **状态感知 (State Awareness)**：让用户一眼看出软件当前是“闲置”、“准备中”还是“工作中”。

## 2. 界面线框图 (Wireframe Description)

## 3. 详细交互与视觉说明

### (A) 核心操作区
*   **布局**：两个并排的大按钮，占据视觉中心。
*   **[🔴 录制]**：
    *   **设计**：白色背景，红色边框，红色图标。
*   **[▶️ 运行]**：
    *   **设计**：**品牌主色填充**（如深蓝色或绿色），白色图标，带阴影（强调它是最常用的按钮）。
    *   **防呆设计**：如果还没有录制过数据，这个按钮置灰不可点，并显示提示“请先教我怎么做”。

### (B) 定时运行
*   **设计**：浅灰色背景的容器，位于底部，视觉权重较低。
*   **交互**：
    *   **开关**：iOS风格的滑块开关（Toggle Switch）。
    *   **反馈**：当开关打开时，下方显示一行小字反馈：*“已激活。哪怕电脑关机，只要您上班解锁屏幕，我就能帮您跑。”* —— **这句话是用户体验的核心，彻底打消用户的“漏跑焦虑”。**

---

## 4. 运行时的反馈设计 (The Feedback)

当用户点击 [运行] 后，软件不能只是默默动鼠标，必须给用户**安全感**。

**屏幕遮罩层 (Screen Mask):**
*   **样式**：全屏覆盖黑色，透明度 10% (微微变暗)。
*   **中央提示框 (HUD)**：
    ```text
    +---------------------------------------+
    |             🤖 正在干活...            |
    |---------------------------------------|
    |  步骤: 正在导出Excel (3/5)            |
    |  ⚠️ 请勿触碰鼠标键盘                  |
    |  [ 按 F12 紧急停止 ]                  |
    +---------------------------------------+
    ```
*   **目的**：明确告知用户“现在我是自动驾驶状态”，防止用户惊慌或误触。

## 5. 给开发的实现 Tip (C# WinForms)

*   **图标**：使用 `FontAwesome` 或 `Segoe MDL2 Assets` 字体图标，保证高清不糊。
*   **圆角**：WinForms 默认是直角，建议重写 `OnPaint` 或使用 `Guna UI` / `ReaLTaiizor` 等免费开源控件库，把按钮和窗口做成 **8px 圆角**，瞬间提升专业度。
*   **颜色规范**：
    *   主色 (Primary): `#007ACC` (VS蓝，专业、冷静)
    *   警告色 (Warning): `#FFC107` (琥珀色)
    *   背景色 (Bg): `#F3F3F3` (干净的灰白)
