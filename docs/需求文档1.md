# 背景
- 需求场景：银行、保险等国央企、公司，各部门的基层职员，每天早上都要做日报，一般耗时5分
钟到1个小时。
- 操作流程：在windows电脑桌面（都是内网，无法连互联网），打开业务系统（公司自研的软件，
类似老浏览器），输入账号密码，点击登录，等待进入（偶尔会弹出通知弹窗，要关闭）；登录后，
点击特定业务、某栏目，点击「昨天」，点击「查询」，点击按钮「导出」，会下载一个excel或
pdf等，然后快捷键回到桌面，打开特定位置的文件，再用快捷键操作，最后完成日报。
- 备用方案：目前的员工都是手动操作；
- 尝试方案：测试，用2个软件实现了上述场景自动化：
    - Tinytask（https://thetinytask.com/）录制用户屏幕操作，保存为exe文件（EXE 
    自带调度逻辑，无需安装任何软件）；
    - system Scheduler（https://www.splinterware.com/products/scheduler.
    html）设定时任务，每天早上定时运行exe文件。

- 痛点：大量员工，懒得装两个软件，懒得学；安装、使用稍微复杂。
    - tinytask 功能、体验很好，软件打开就能用，很简单；但 system Scheduler有点重，
    还得安装，功能冗余；
- 目标：计划开发一个轻量、简单、可用、易用、用户体验好的软件，实现上述需求。


## 2.1 任务数据 (`task.json`)
*存储于 EXE 同级目录。*

```json
{
  "meta": {
    "version": "1.0",
    "created_at": 1716300000,
    "resolution": "1920x1080", // 仅记录，v1.0 不做自动缩放适配
    "total_events": 150
  },
  "events": [
    {
      "type": "mouse_move",    // 枚举: mouse_move, mouse_click, key_press
      "x": 1024,               // 屏幕绝对坐标 (Absolute Screen Coordinates)
      "y": 768,
      "button": "left",        // left, right, middle, double, none
      "key_code": 0,
      "delay": 50              // 距离上一动作的毫秒数 (Delta Time)
    },
    {
      "type": "key_press",
      "key_code": 13,          // Virtual Key Code (e.g., VK_RETURN)
      "delay": 200
    }
  ]
}
```
