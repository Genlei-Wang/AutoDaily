
# 背景
- 需求场景：银行、保险等国央企、公司，各部门的基层职员，每天早上都要做日报，一般耗时5分钟到1个小时。
- 操作流程：在windows电脑桌面（都是内网，无法连互联网），打开业务系统（公司自研的软件，类似老浏览器），输入账号密码，点击登录，等待进入（偶尔会弹出通知弹窗，要关闭）；登录后，点击特定业务、某栏目，点击「昨天」，点击「查询」，点击按钮「导出」，会下载一个excel或pdf等，然后快捷键回到桌面，打开特定位置的文件，再用快捷键操作，最后完成日报。
- 备用方案：目前的员工都是手动操作；
- 尝试方案：测试，用2个软件实现了上述场景自动化：
    - Tinytask（https://thetinytask.com/）录制用户屏幕操作，保存为exe文件（EXE 自带调度逻辑，无需安装任何软件）；
    - system Scheduler（https://www.splinterware.com/products/scheduler.html）设定时任务，每天早上自动运行exe文件。

- 痛点：大量员工，懒得装两个软件，懒得学；安装、使用稍微复杂。
    - tinytask 功能、体验很好，软件打开就能用，很简单；但 system Scheduler有点重，还得安装，功能冗余；
- 目标：计划开发一个轻量、简单、可用、易用、用户体验好的软件，实现上述需求。

## 2. 交互

### (A) 录制、运行操作区
*   **布局**：两个并排的大按钮，占据视觉中心。
*   **[🔴 录制]**：
    *   **设计**：红色边框，红色图标。
*   **[▶️ 运行]**：
    *   **设计**：蓝色边框，蓝色图标。
    *   **防呆设计**：如果还没有录制过数据，这个按钮置灰不可点，点击后提示“请先录制操作哦”。

### (B) 定时运行
*   **设计**：浅灰色背景的容器，位于底部。
*   **交互**：
    *   **开关**：iOS风格的滑块开关（Toggle Switch）。
    *   **反馈**：当开关打开时，下方显示一行小字反馈：*“已开启定时运行”*；然后展示定时配置（默认9点），下次运行时间。

## 3. 运行时的反馈设计 (The Feedback)

**屏幕提示:**
*   **小提示框 (HUD)**：
    ```text
    +---------------------------------------+
    |             🤖 正在操作...            |
    |---------------------------------------|
    |  步骤: 具体类型，如点击、输入、滚动等 (3/5) |
    |  ⚠️ 请勿触碰鼠标键盘                  |
    |  [ 按 F12 紧急停止 ]                  |
    +---------------------------------------+
    ```
*   **目的**：明确告知用户“现在我是自动驾驶状态”，防止用户惊慌或误触。
